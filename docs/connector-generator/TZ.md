# Генератор символов разъемов KiCad

## Цель

Сделать автоматизацию для регулярных разъемов в `Ki-Connectors.kicad_sym`, чтобы программа
выполняла повторяемую работу без ручных ошибок: создание рядов, пинов, нумерации и базовой
графики символа.

Генератор должен работать с обычным текстовым `.kicad_sym` файлом. Git остается основным
механизмом контроля изменений: перед записью можно смотреть diff, а ошибочный результат
легко откатить.

## Базовая идея

KiCad остается редактором сложной графики, а генератор отвечает за регулярную часть:

- определить покрываемые символы по явному списку в конфиге;
- пересоздать базовый generated body style;
- расставить пины и номера по выбранной схеме;
- сохранить ручные альтернативные варианты внутри символа;
- проверить отсутствие дублирования и пропусков в нумерации.

## Область ответственности генератора

Генератор может менять только покрываемые символы, указанные в конфиге.

Внутри покрываемого символа генератор владеет только базовым body style `1`:

```text
Con-2X-10P-S_1_1  generated
Con-2X-10P-S_2_1  generated
Con-2X-10P-S_1_2  manual, сохранить
Con-2X-10P-S_2_2  manual, сохранить
```

Все вложенные блоки с body style `2`, `3` и далее считаются ручными альтернативами и
переносятся в новый символ без изменения.

## Именование

Стартовая грамматика для регулярных разъемов:

```text
Con-<rows>X-<pins_per_row>P-<scheme>
Con-1X-04P-R
Con-2X-10P-R
Con-2X-10P-S
Con-3X-08P-C
```

Где:

- `rows` - количество рядов;
- `pins_per_row` - количество позиций в одном ряду;
- `scheme` - схема нумерации.

Пока символы берутся из конфига, а не по glob-шаблону имени. Это сохраняет контроль:
похожее имя не означает автоматическое изменение.

Для массовой генерации количество пинов в ряду может задаваться отдельно по рядности:

```toml
[series.pins_per_row_by_rows]
"1" = [1..40]
"2" = [2..40]
"3" = [3..40]
```

Старый общий `pins_per_row = [...]` остается fallback для простых серий.

## Схемы нумерации

Минимальный набор:

```text
row
  row A: 1..N
  row B: N+1..2N
  row C: 2N+1..3N

column
  column 1: row A=1, row B=2, row C=3
  column 2: row A=4, row B=5, row C=6

column_snake
  column 1: A,B,C
  column 2: C,B,A
  column 3: A,B,C

custom
  явная таблица номеров для нестандартных случаев
```

Для текущего `Con-2X-10P-S` базовая ожидаемая схема:

```text
unit 1: 1,3,5...19
unit 2: 2,4,6...20
```

## Шаблоны и вставляемые блоки

Целевая архитектура должна поддерживать template-expander:

- сложные элементы рисуются в KiCad как образцы;
- генератор клонирует готовые S-expression блоки;
- генератор умеет сдвигать блоки, размножать клетки и заменять номера пинов.

Это позволит делать шапки таблиц, стрелки, подписи и другие элементы не через описание всех
возможностей KiCad в YAML/TOML, а через готовые KiCad-фрагменты.

Первый рабочий этап может иметь встроенный базовый renderer для простого прямоугольного
разъема. Следующий этап - вынести графику в template `.kicad_sym`.

## Базовый стиль прямоугольного разъема

За ориентир оформления берется существующий `Con-2X50P(100P)-XS`:

- `Reference` и `Value` видимые;
- `pin_names` скрыты;
- `pin_numbers` включены, как галочка "Показать номер вывода" в свойствах символа;
- внутри прямоугольника есть левая колонка нумерации;
- прямоугольник может иметь заливку `background`, как в ручных `Con-1X-01P-R` и `Con-1X-02P-R`;
- левая колонка отделена вертикальной линией `x=5`;
- номера выводятся отдельными `text`-элементами, а не только через свойства пинов;
- пины остаются настоящими `pin` с тем же номером и именем, чтобы KiCad сохранял электрическую модель.

Ручные альтернативные body style продолжают сохраняться без изменения.

В `.kicad_sym` KiCad хранит скрытие номеров как `(pin_numbers (hide yes))`.
Для включенного "Показать номер вывода" generated-символ не должен содержать
`pin_numbers hide`.

## Проверки

Перед записью генератор должен уметь:

- показать diff;
- проверить, что количество пинов равно `rows * pins_per_row`;
- проверить уникальность pin number внутри generated body style;
- проверить отсутствие пропусков для числовой последовательности `1..N`;
- сохранить ручные body style без изменения;
- завершиться с ошибкой в режиме `--check`, если библиотека не соответствует конфигу.

## Первые задачи

1. Добавить конфиг покрываемых символов.
2. Реализовать безопасное чтение `.kicad_sym` и замену только выбранных top-level symbols.
3. Реализовать генерацию базового body style `1` для 1/2/3 рядов.
4. Сохранять ручные альтернативы `_N_2`, `_N_3` и далее.
5. Добавить unit-тесты на нумерацию и сохранение ручных альтернатив.
6. Позже добавить template-expander для вставляемых KiCad-блоков.

## GUI-сценарий

Плагин KiCad должен быть ориентирован на человека, а не на чтение S-expression diff.
Основной экран:

- параметры серии: префикс, ряды, список пинов в ряду, схемы нумерации;
- список символов, которые будут созданы или обновлены;
- статус по каждому символу: создать, обновить, сохранить ручные варианты;
- предпросмотр нумерации выбранного символа в виде таблицы пинов;
- кнопка генерации.

Технический diff остается доступным как дополнительная диагностика, но не является
основным способом проверки результата.
